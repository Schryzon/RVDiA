# Contributing to RVDiA

Thank you for your interest in contributing to **Revolutionary Virtual Digital Assistant (RVDiA)**! We welcome contributions from everyone, whether it's fixing bugs, adding new features, or improving documentation.

## How to Contribute

### 1. Reporting Bugs
- Use the [GitHub Issues](https://github.com/Schryzon/RVDiA/issues) to report bugs.
- Provide a clear and descriptive title.
- Describe the exact steps to reproduce the issue.
- Include screenshots or error logs if possible.

### 2. Suggesting Enhancements
- Check the issues list to see if the feature has already been suggested.
- Open a new issue with the tag `enhancement`.
- Explain why this feature would be useful for the bot.

### 3. Pull Requests
- Fork the repository.
- Create a new branch for your feature or bugfix (`git checkout -b feature/my-new-feature`).
- Ensure your code follows the existing style (Snake_case, clean indentation).
- Commit your changes with clear messages.
- Push to your fork and submit a Pull Request.

## Development Setup
1. Clone the repository.
2. Install dependencies: `pip install -r requirements.txt`.
3. Set up your `.env` file (see `README.md`).
4. Run the bot: `python312 RVDIA.py`.

## Coding Guidelines
- **Naming**: Use `snake_case` for variables and functions.
- **Logic**: Prefer flat logic over deep nesting. Use early returns.
- **Safety**: Don't overprotect code. Let it fail clearly during development.
- **Comments**: Keep comments honest and raw.
